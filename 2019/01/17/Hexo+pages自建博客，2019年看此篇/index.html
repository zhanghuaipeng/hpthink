Hexo+pages自建博客，2019看此篇

发表于 2019-01-17 | 分类于 hexo |
字数统计: | 阅读时长 ≈

本文以windows为例，来说明如何利用Hexo+pages自建博客。大约1-2两小时，您就可以拥有自己的博客。

为什么要自建博客

个性化，你可以使用所有的官方主题：https://hexo.io/themes/ ，并且有丰富的插件来个性化自己的博客。
还不够？那还可以绑定域名，比如通过自己姓名的全拼来访问博客；另外，所有的博客文章以MD的形式在你本地都会有一份。

node.js：https://nodejs.org/en/download/
git：https://git-scm.com/book/en/v2/Getting-Started-Installing-Git

站内内容

新建好的文件夹目录如下：

_config.yml

网站的 配置 信息，您可以在此配置大部分的参数。

package.json

应用程序的信息。EJS, Stylus 和 Markdown renderer 已默认安装，您可以自由移除。

scaffolds

模版 文件夹。当您新建文章时，Hexo 会根据 scaffold 来建立文件。

Hexo的模板是指在新建的markdown文件中默认填充的内容。例如，如果您修改scaffold/post.md中的Front-matter内容，那么每次新建一篇文章时都会包含这个修改。

==source==

资源文件夹是存放用户资源的地方。_posts,我们所有自己写的文章，都要以MD的格式，放在此文件夹下面。除 _posts 文件夹之外，开头命名为 _ (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。

themes

主题 文件夹。Hexo 会根据主题来生成静态页面。
